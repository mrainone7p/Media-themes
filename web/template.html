PIPELINE STEPS
Choose which parts of the automated pipeline should run when the Scheduler configuration is triggered.
Step 1 Scan Libraries
Connects to Plex, indexes media, and checks whether each folder already has a local theme file. Items without a theme are marked Missing. Items with a theme are marked Available. Uses Plex only. Fast.
Step 2 Find Theme Sources
Searches for theme sources for each Missing item using the configured search logic. When a source is found, the item moves to Staged for manual review and Approval. Items marked Unmonitored or Failed are ignored. Uses TMDB, YouTube, or Golden Source when configured.
Only use Golden Source
Use a curated Golden Source for tighter, faster, and more reliable source discovery.
Missing Titles Per Run
Limits how many Missing items are processed per run. 0 = unlimited.
Step 3 Download Themes
Downloads themes for each Approved item, saves the theme file into the media folder, and moves the item to Available. Items marked Unmonitored or Failed are ignored. Uses yt-dlp and ffmpeg.
Auto-Approve Staged Items
Automatically move Staged items to Approved before downloading for a fully hands-off pipeline.

How the pipeline works
Three steps process your media library from discovery to download.
Step 1
⟳ Scan Libraries
Connects to Plex, checks each media folder for a local theme file, and marks items Missing or Available.
Step 2
⚡ Find Sources
Searches for theme sources for Missing items using TMDB, YouTube, or Golden Source. Matches move to Staged.
Step 3
↓ Download Approved
Downloads theme files for Approved items using yt-dlp and ffmpeg, saves them to the media folder, and marks them Available.
Delete Theme
This will permanently delete the theme file from disk. The media item will be set back to Missing status.
